第二步：安装hexo

  1. 创建一个文件夹用于存放你所有需要放在自己blog资料

  2. 在文件夹中，右键选择并点击Git Bash Here

  3. 在命令行窗口输入，安装Hexo 命令行工具的包名 ：

  • npm 是 Node.js 包管理器的命令行工具，用于安装和管理 JavaScript 包和依赖项。
    • install 是 npm 命令的子命令，用于安装包。
    • hexo-cli 是要安装的包的名称。这是 Hexo 命令行工具的包名。
    • -g 是一个标志，表示要将包全局安装。这意味着 Hexo 命令行工具将在你的计算机上全局可用，而不仅仅是在当前项目中可用。

（运行这个命令将会在你的计算机上全局安装 Hexo 命令行工具，使你能够在命令行中使用 Hexo 的各种命令来管理和生成静态博客网站）

第三步：对Hexo进行配置

  1. 初始化：

    这个命令帮助你快速开始一个新的 Hexo 博客项目，并为你创建了必要的文件和结构。一旦你运行了 hexo init，你可以开始编写文章、配置博客选项，然后使用 Hexo 命令生成静态网站并发布它。

  2. 下面是经常会用到的本地步骤：

    hexo clean 主要删除 Hexo 项目中以下两个位置的文件和文件夹：

    1. _public 文件夹（默认情况下）：这个文件夹包含了通过 hexo generate 命令生成的静态网站文件，包括 HTML、CSS、JavaScript 等。运行 hexo clean 将删除这个文件夹及其内容。`
    2. .hexo 文件夹：这个文件夹包含了 Hexo 内部的缓存文件，用于加速生成过程。运行 hexo clean 会清除这些缓存文件。

    这两个位置的文件和文件夹都是由 Hexo 自动生成和管理的，通常不需要手动编辑或删除它们。运行 hexo clean 主要是为了在需要重新生成静态网站时清理旧的文件和缓存，以确保生成的网站是最新的版本。所以，在执行 hexo clean 命令之前，请确保你不再需要之前生成的静态网站文件，因为它们将被永久删除。

    hexo generate（或者可以用缩写形式 hexo g）是 Hexo 命令行工具的一个命令，用于生成静态网站文件。

    当你运行 hexo generate 命令时，Hexo 将会执行以下操作：

    1. 使用你的 Hexo 配置文件（通常是 _config.yml）中的设置来渲染和生成静态网站的页面。

    2. 将 Markdown 文件转换为 HTML 格式，应用主题和布局，生成最终的静态网页。

    3. 将生成的静态网站文件保存在配置文件中指定的输出目录中，通常是 _public 文件夹（可以在配置文件中自定义）。

    4. 创建站点地图（sitemap）和 RSS 订阅（如果在配置文件中启用了这些功能）。

      hexo generate 是构建 Hexo 博客网站的关键步骤之一。一旦你运行了这个命令，你就会得到一个包含所有生成的静态页面的文件夹，你可以将这些文件部署到网络服务器上，以便将你的博客发布到互联网上。在执行 hexo generate 前，通常需要确保你的博客配置文件和文章内容都已准备好。

    hexo s（或者可以用缩写形式 hexo server）是 Hexo 命令行工具的一个命令，用于启动本地服务器以预览你的博客网站。

    当你运行 hexo s 命令时，Hexo 将会执行以下操作：

    1. 读取你的 Hexo 配置文件（通常是 _config.yml）中的设置。
    2. 在本地计算机上启动一个轻量级的 Web 服务器。
    3. 将你的博客网站在本地服务器上提供出来，可以通过网址访问，通常是 http://localhost:4000。
    4. 当你在浏览器中访问这个网址时，你可以实时预览你的博客网站，包括文章、页面和样式。

    hexo s 命令通常用于在本地开发和测试博客网站，以便你可以在发布到互联网之前查看你的更改。它提供了一个方便的方式来查看你的博客的外观和功能，而不必上传到实际的网络服务器上。要停止本地服务器，只需按下 Ctrl + C 键组合（在大多数操作系统中），然后选择是否停止服务器。

    总之，hexo s 是一个方便的命令，用于本地预览和测试你的 Hexo 博客网站。

  • 注意：因为hexo是静态博客，所以每次对博客做出改变后都需要清除原来的缓存重新生成一次

  1. 将制作好的网页部署到网页端（安装+部署）：

    npm install hexo-deployer-git --save 是一个用于在 Hexo 项目中安装并保存依赖的命令。

    具体来说：

    • npm install 是使用 Node.js 包管理器（npm）来安装包的命令。

    • hexo-deployer-git 是要安装的包的名称，它是 Hexo 用于通过 Git 将博客网站部署到远程仓库的插件。这个插件允许你使用 Git 来发布你的博客。

    • --save 是一个标志，它在安装包的同时将包的信息添加到项目的 package.json 文件中的 dependencies 部分，以便其他开发者在克隆你的项目时可以自动安装相同的依赖。

      运行这个命令后，Hexo 将会安装 hexo-deployer-git 插件并将其添加到你的 Hexo 项目的依赖中。这样，你就可以配置并使用该插件来部署你的博客到 Git 仓库中，使其在网络上可访问。

    hexo d（或者可以用缩写形式 hexo deploy）是 Hexo 命令行工具的一个命令，用于将你的博客网站部署到远程服务器或托管服务上。

    当你运行 hexo d 命令时，Hexo 将会执行以下操作：

    1. 读取你的 Hexo 配置文件（通常是 _config.yml）中的设置，包括部署配置。
    2. 根据配置中的信息，将生成的静态网站文件上传到远程服务器或托管服务，如GitHub Pages、Netlify、Vercel等。
    3. 部署完成后，你的博客网站将在远程服务器或托管服务上可访问，用户可以通过互联网浏览你的博客内容。

    请注意，使用 hexo d 命令进行部署前，你需要先配置好 Hexo 的部署选项，包括目标远程服务器或托管服务的信息，认证凭据等。这些配置通常在 Hexo 的配置文件中进行设置。不同的部署选项可能需要不同的配置，因此请查阅相关文档以确保配置正确。

    总之，hexo d 命令是用于将你的 Hexo 博客网站部署到远程服务器或托管服务的命令，以便让其他人通过互联网访问你的博客。

第二章：装主题butterfly

  1. 在hexo项目的根目录下面下载主题

  2. 安装pug 和 stylus 渲染器

  3. 修改项目根目录下的_config.yml文件（称为站点配置文件），开启主题。

  4. 升级建议

    为了減少升级主题带来的不便，我们可以把主题文件夹中的 _config.yml 重命名为 _config.butterfly.yml，复制到 Hexo 根目录下与_config.yml同级。

    Hexo会自动合并主题中的_config.yml和 _config.butterfly.yml ，如果存在同名配置，会使用_config.butterfly.yml的配置，其优先度较高。所以像和博客网址相关联的固定资料可以设置在_config.yml中，比如博客的标题、作者信息和邮箱等等资料，而和主题样式相关的配置放在 _config.butterfly.yml 中，那么在将来你想换一个主题是很方便的。
